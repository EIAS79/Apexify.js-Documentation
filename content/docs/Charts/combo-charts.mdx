# Combo charts (bar + line)

**Combo charts** overlay **column bars** and **one or more line series** on the **same continuous X domain**. Bars always use the **primary (left) Y-axis**. Lines default to a **secondary (right) Y-axis** so you can compare volume (bars) with rates or percentages (lines)—typical for dashboards.

They are created with **`ApexPainter.prototype.createComboChart(options)`** (`ChartCreator.createComboChart`), **not** `createChart(...)`.

## Overview

| Concept | Details |
|--------|---------|
| **Entry point** | `await painter.createComboChart({ bars, lines, ... })` → `Promise<Buffer>` |
| **Bar geometry** | Same **`BarChartData`** as [Bar charts](./bar-charts#data-structure): `value` + `xStart`/`xEnd` for **standard** columns, or `values` + `barsType` for **grouped** / **stacked** segments. |
| **Line series** | **`ComboLineSeries`**: extends [Line chart](./line-charts) `LineSeries` with optional **`yAxis: 'primary' \| 'secondary'`** (default **`secondary`**) and optional **`opacity`** (multiplies global `opacity.lines`). |
| **X alignment** | Bar positions and line **`data[].x`** share one **continuous X scale**; set [axis tick visibility](./bar-charts#axes-configuration) if you only want category labels from bars. |
| **Background** | **`options.appearance`** — same **`ChartAppearanceExtended`** pipeline as other charts (`backgroundGradient`, `patternBg`, `noiseBg`, `axisColor`, …). |

## Bar modes: `barsType`

| `barsType` | Bar data | Behavior |
|------------|----------|----------|
| **`standard`** (default) | Each row uses **`value`** + **`xStart`** / **`xEnd`** | Classic columns. |
| **`grouped`** | Each row uses **`values`** array (segments with colors/labels) | Side‑by‑side segments per category X slot. |
| **`stacked`** | Same **`values`** shape | Segments stacked vertically; totals align with primary Y. |

`barsType` is ignored unless bars define **`values`** (multi-segment rows).

## Secondary Y-axis

- **`axes.y`** — primary scale (**bars** + any line with **`yAxis: 'primary'`**).
- **`axes.ySecondary`** — secondary scale on the **right** (lines with **`yAxis: 'secondary'`** or default line assignment).
- **`secondaryYAxis.show`** — optional override; by default the right axis appears when any line uses the secondary scale.

Axis **`AxisConfig`** supports **`label`**, **`labelColor`**, **`color`** (tick/spine ink), **`range`**, **`tickFontSize`**, etc., same as line charts. On **dark** canvases, set **light** `labelColor` / `color` for readable ticks (see [Bar charts — axes](./bar-charts#axes-configuration)).

## Layer opacity

**`options.opacity`** scales entire layers (each multiplier **0–1**):

- **`bars`**, **`lines`**, **`grid`**
- **`primaryAxis`**, **`secondaryAxis`**, **`xAxis`**
- **`chartTitle`**, **`legend`**, **`barLabels`**, **`valueLabels`**, **`pointLabels`**

Useful for de‑emphasizing grid or axes without changing colors.

## Legend

Combo charts use the same **`legend`** shape as bar/line charts: **`position`** (`LegendPlacement`), **`entries`**, **`textColor`**, **`backgroundColor`**, **`borderColor`**, **`wrapText`**, **`maxWidth`**, **`fit`** (`content` | `stretch`), etc. On dark backgrounds, set a **dark semi‑opaque** `backgroundColor` and a **light** `textColor` so labels stay readable.

## Basic example — standard bars + line on secondary axis

<ts>
```typescript
import { ApexPainter } from 'apexify.js';
import fs from 'fs';

const painter = new ApexPainter();

const buf = await painter.createComboChart({
  barsType: 'standard',
  dimensions: { width: 880, height: 560 },
  appearance: {
    backgroundGradient: {
      type: 'linear',
      startX: 0,
      startY: 0,
      endX: 880,
      endY: 560,
      colors: [
        { stop: 0, color: '#0f172a' },
        { stop: 1, color: '#020617' },
      ],
    },
    axisColor: '#e2e8f0',
  },
  axes: {
    x: { label: 'Quarter', labelColor: '#f8fafc', range: { min: 0, max: 8 }, color: '#e2e8f0', tickFontSize: 12 },
    y: { label: 'Volume (k)', labelColor: '#f8fafc', range: { min: 0, max: 100 }, color: '#e2e8f0', tickFontSize: 12 },
    ySecondary: {
      label: 'Margin %',
      labelColor: '#fdf4ff',
      range: { min: 40, max: 90 },
      color: '#fda4af',
      tickFontSize: 12,
    },
  },
  bars: [
    { label: 'Q1', value: 62, xStart: 1, xEnd: 2, color: '#38bdf8' },
    { label: 'Q2', value: 74, xStart: 3, xEnd: 4, color: '#a78bfa' },
  ],
  lines: [
    {
      label: 'Margin',
      color: '#fb7185',
      lineWidth: 2,
      smoothness: 'bezier',
      yAxis: 'secondary',
      data: [
        { x: 1.5, y: 52 },
        { x: 3.5, y: 61 },
      ],
    },
  ],
  labels: {
    title: { text: 'Combo · bars + dual-axis line', fontSize: 18, color: '#f8fafc' },
  },
  legend: {
    show: true,
    position: 'top-left',
    fontSize: 12,
    textColor: '#f8fafc',
    backgroundColor: 'rgba(15, 23, 42, 0.96)',
    borderColor: 'rgba(226, 232, 240, 0.42)',
    padding: 10,
  },
});

fs.writeFileSync('combo.png', buf);
```

## Grouped bars + lines

Use **`barsType: 'grouped'`** and **`values`** on each bar row (same segment model as [Bar charts — grouped](./bar-charts#chart-types)). Lines still use **`ComboLineSeries`**; align **`x`** with your category bands.

## API surface (TypeScript)

The full **`ComboChartOptions`** interface lives in **`apexify.js`** at `Canvas/utils/Charts/combochart` (also re-exported via package typings). Key groups:

- **`bars`**, **`lines`**, **`barsType`**, **`dimensions`**, **`appearance`**, **`axes`**, **`opacity`**, **`labels`**, **`legend`**, **`grid`**, **`barStyle`**, **`secondaryYAxis`**
