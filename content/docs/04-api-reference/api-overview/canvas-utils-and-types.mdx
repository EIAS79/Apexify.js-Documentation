## Types barrel (`CanvasTypes`)

Primary contracts ship as **`CanvasTypes`** from **`apexify.js`** (large barrel — your editor “go to definition” is the practical map):

- Scene: **`CanvasConfig`**, **`BackgroundLayer`**, **`PatternOptions`**
- Raster layers: **`ImageProperties`**, **`CreateImageOptions`**, **`GroupTransformOptions`**, **`ImageFilter`**
- Text: **`TextProperties`**, **`TextObject`** (legacy alias)
- GIF / batch / stitch / palette / mask / blend / animation frames — see import hints inside the file

**Naming traps**

- **`cropOptions`** vs **`CropOptions`** — different structs for different entry points.
- **`VideoCreationOptions`** — declared beside the internal video creator (infer **`Parameters<ApexPainter['createVideo']>[0]`** when barrels disagree) — [`createVideo` router](/docs#video-create-video), [Video router internals](/docs#internals-video-router).
- **`CanvasResults`** — defined near **`CanvasCreator`**.

Import types from **`apexify.js`** the same way as runtime helpers when re-exported.
