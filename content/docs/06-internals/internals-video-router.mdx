# Video router internals

## Types location

**`VideoCreationOptions`**, **`MixAudioOperation`**, **`MixAudioOverlayClip`** are declared in **`lib/Canvas/extended/VideoCreator.ts`** (re-exported from **`extended/index.ts`**). They may not appear on every root **`apexify.js`** type barrel — use **`Parameters<ApexPainter['createVideo']>[0]`** or copy shapes locally when needed.

## Branch dispatch

The **`createVideo`** implementation is a **top-to-bottom `if` chain**. The **first** truthy operation group **wins**; later keys are ignored. Example: **`getInfo: true`** together with **`convert`** returns **only** metadata — **`convert`** never runs.

## Prefetch behavior

When **`getInfo`**, **`extractFrame.frame`**, **`generateThumbnail`**, or **`generatePreview`** is used, **`getVideoInfo`** may run once up front (**skip FFmpeg check** path) to supply **`videoInfo`** to thumbnail/preview helpers inside **`VideoCreator`**.

## HTTP sources vs helpers

Some **`ApexPainter`** code paths resolve URLs inconsistently compared to **`VideoHelpers.resolveVideoSource`** — prefer **buffers** or **local files** for reliability.
