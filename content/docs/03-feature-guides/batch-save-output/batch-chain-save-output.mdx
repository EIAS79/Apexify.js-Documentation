# Batch, chain, save & output

Run **parallel** canvas/image/text jobs (**`batch`**), pipe **buffers through sequential `ApexPainter` calls** (**`chain`**), **serialize** PNGs (**`outPut`**), and **write files** with naming helpers (**`save`**, **`saveMultiple`**).

---

## Constructor default for `outPut`

```typescript
new ApexPainter({ type: 'buffer' }) // default — outPut returns Buffer unchanged
```

**`type`** may be **`'buffer'`** | **`'base64'`** | **`'dataURL'`** | **`'blob'`** | **`'url'`** | **`'arraybuffer'`**. This controls **`outPut()`** only — **`createCanvas`** and friends still return raw **`Buffer`** unless you wrap them.

---

## Quick start

<CodeSwitcher>
<ts>
```typescript
import { ApexPainter } from 'apexify.js';

const painter = new ApexPainter();

const layers = await painter.batch([
  { type: 'canvas', config: { width: 320, height: 200, colorBg: '#0ea5e9' } },
  { type: 'canvas', config: { width: 320, height: 200, colorBg: '#6366f1' } },
]);

const meta = await painter.saveMultiple(layers, {
  directory: './renders',
  naming: 'counter',
  prefix: 'slide-',
  format: 'png',
});

console.log(meta.map((m) => m.path));
```
</ts>
<js>
```javascript
const { ApexPainter } = require('apexify.js');

(async () => {
  const painter = new ApexPainter();
  const layers = await painter.batch([
    { type: 'canvas', config: { width: 320, height: 200, colorBg: '#0ea5e9' } },
    { type: 'canvas', config: { width: 320, height: 200, colorBg: '#6366f1' } },
  ]);
  const meta = await painter.saveMultiple(layers, {
    directory: './renders',
    naming: 'counter',
    prefix: 'slide-',
    format: 'png',
  });
  console.log(meta.map((m) => m.path));
})();
```
</js>
</CodeSwitcher>

## `validHex`

**`painter.validHex('#RRGGBB')`** returns **`true`** or **throws** — useful before feeding hex into configs.

---

## Hosted URL via **`outPut`** (**`type: 'url'`**)

You can turn PNG **`Buffer`**s into a **hosted HTTPS link** for quick sharing by setting **`new ApexPainter({ type: 'url' })`** and **`await painter.outPut(buffer)`**. See [Save & output](/docs#save-and-output) for examples.

That path uses a **bundled prototype uploader** — fine for demos; production stacks usually substitute their own storage. Module-level detail: [Batch & image I/O internals](/docs#internals-batch-image-io).
