# Professional image filters (`ImageFilter[]`) {#05-create-image-filters}


The **`filters`** field is a **stack** of **`ImageFilter`** objects. Filters are applied with **Sharp** (and a pixel path when **`grayscale`** is included). They operate on **bitmap** layers only (not on pure shape vector fills in the same way — shape layers apply filters on the **clipped rectangular** region before the shape is drawn).

---

## Global tuning

| Field | Type | Default | Role |
|-------|------|---------|------|
| **`filterOrder`** | **`'pre' \| 'post'`** | **`'post'`** | **Pre** draws the fitted image to a temp surface, filters it, then pastes it (skips mask, distortion, mesh warp, **post** filters, **effects**, and **stroke** — see [Paint order](/docs#09-create-image-paint-order)). **Post** filters the pixels already painted **inside the clip**. |
| **`filterIntensity`** | **`number`** | **`1`** | Multiplier applied to each filter’s **`intensity`**, **`value`**, and **`radius`** when building the adjusted stack. |

```typescript
await painter.createImage(
  {
    source: './photo.jpg',
    x: 0,
    y: 0,
    width: 640,
    height: 480,
    fit: 'cover',
    filters: [
      { type: 'gaussianBlur', intensity: 0.15 },
      { type: 'brightness', value: 0.08 },
    ],
    filterIntensity: 1.2,
    filterOrder: 'post',
  },
  buffer
);
```

## Filter catalogue (`ImageFilter.type`)

Unless noted, omit **`intensity`** / **`value`** or use **`0`** where the implementation skips work.

### Blur & sharpen

| `type` | Main fields | Behaviour |
|--------|-------------|-----------|
| **`gaussianBlur`** | **`intensity`** | Gaussian blur strength |
| **`motionBlur`** | **`intensity`**, **`angle`** (deg, default **`0`**) | Directional blur |
| **`radialBlur`** | **`intensity`**, **`centerX`**, **`centerY`** | Spin blur around centre (defaults to bitmap centre in pixel space) |
| **`sharpen`** | **`intensity`** | Sharpening |

### Colour & tone

| `type` | Main fields | Behaviour |
|--------|-------------|-----------|
| **`brightness`** | **`value`** | Positive / negative adjustment |
| **`contrast`** | **`value`** | Contrast adjustment |
| **`saturation`** | **`value`** | Saturation adjustment |
| **`hueShift`** | **`value`** | Hue rotation |
| **`grayscale`** | — | Forces a **pixel-based** filter path if mixed with others |
| **`sepia`** | — | Sepia tone |
| **`invert`** | — | Colour inversion |
| **`posterize`** | **`levels`** (default **`4`**) | Posterisation |

### Stylisation

| `type` | Main fields | Behaviour |
|--------|-------------|-----------|
| **`pixelate`** | **`size`** (default **`10`**) | Mosaic blocks |
| **`noise`** | **`intensity`** (**`0…1`** clamped) | Noise overlay |
| **`grain`** | **`intensity`** (**`0…1`** clamped) | Film-like grain |
| **`edgeDetection`** | **`intensity`** | Edge emphasising |
| **`emboss`** | **`intensity`** | Emboss |

---

## Combining filters

- Order matters: filters run **first → last** as listed.
- **`grayscale`** triggers an alternate pixel pipeline for the whole stack (still synchronous from your **`await createImage`** perspective).
- Prefer **`filterOrder: 'post'`** when you need **`stroke`**, **`effects`**, **`mask`**, or **`distortion`** in the same layer — **`pre`** exits before those stages.

---

## Shape layers

When **`source`** is a **`ShapeType`**, **`filters`** are applied inside **`drawShape`** on the **`width × height`** clip region **before** painting the vector shape. **`filterOrder`** and **`filterIntensity`** from **`ImageProperties`** are **not** applied on that path — pass concrete **`intensity`** / **`value`** on each **`ImageFilter`**.
