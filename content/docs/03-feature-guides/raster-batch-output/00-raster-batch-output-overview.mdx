# Raster, batch & output {#00-raster-batch-output-overview}

This section is the **single home** for **`lib-next`** workflows that turn **`Canvas`** **`/`** **`Buffer`** results into files **`/`** encoded strings, run **parallel batch** or **sequential chain** pipelines, manipulate **pixels** **`/`** **paths** **`/`** **hit regions**, and call **`painter.image`** helpers (**stitch**, **collage**, **Sharp** compression, palette, resize, blend, …).

**Composition vs utilities:** layering assets with **`createImage`** **`/`** **`createCanvas`** is documented under **images **`/`** shapes**. **`painter.image`** (**whole-file **`/`** **`Buffer`** helpers — resize, stitch, compress, blend, …) is **[§04–§07](#guide-index)** in this folder only.

**Implementation map**

| Area | Code |
|------|------|
| Batch **`/`** chain | **`lib-next/batch/batch-operations.ts`**, **`apex-painter/creates/batch-create.ts`** |
| Save **`/`** **`outPut`** | **`lib-next/output/save-buffer.ts`**, **`buffer-output.ts`**, **`apex-painter/creates/output-save.ts`** |
| **`painter.output`** | **`apex-painter/facets.ts`** · **`output/buffer-encoding.ts`**, **`upload-imgur.ts`** |
| **`painter.image`** | **`lib-next/image/painter-image-utils.ts`**, **`output/stitch.ts`**, **`output/compression.ts`** |
| **`painter.pixels`** | **`pixels/pixel-data-creator.ts`** |
| **`painter.path2d`** | **`path/path2d-creator.ts`**, **`apex-painter/path-custom-lines.ts`** (**`custom`**) |
| **`painter.detect`** | **`pixels/hit-detection-creator.ts`** |
