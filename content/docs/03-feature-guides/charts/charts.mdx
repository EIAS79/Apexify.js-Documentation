## What you get

Each chart API resolves to a **PNG `Buffer`** — same shape as **`createCanvas`** / **`createText`** results.

---

## What charts are for

Charts turn **tabular data** into ready-made PNG graphics so you do not place every bar or slice manually. **`ApexPainter`** exposes **`createChart`**, **`createComparisonChart`**, and **`createComboChart`**. You can also call **`Charts.create*`** from **`apexify.js`** without constructing a painter when that fits your app structure.

## Facade methods (summary)

| Method | Role |
|--------|------|
| **`createChart(type, data, options?)`** | Dispatches to pie, bar, horizontal bar, line, scatter, radar, polar area |
| **`createComparisonChart(options)`** | Two charts in one canvas (**`sideBySide`** \| **`topBottom`**); panels may mix kinds (**`donut`** only here, not in **`createChart`**) |
| **`createComboChart(options)`** | Bars + lines; optional **secondary Y**; **`barsType`** **`standard`** \| **`grouped`** \| **`stacked`** |

**Supported `createChart` literals:** `'pie'`, `'bar'`, `'horizontalBar'`, `'line'`, `'scatter'`, `'radar'`, `'polarArea'`.

## Chart modules (one file per family)

| Module | Notes |
|--------|------|
| **`piechart`** | Slices, donut radius, standard vs connected legend, slice stroke/shadow/**opacity** |
| **`barchart`** | Vertical bars, **`BarChartType`**, waterfall optional, dual axes, stacked/clustered/lollipop/dot modes |
| **`horizontalbarchart`** | Horizontal layout + legend inset rules |
| **`linechart`** | Lines, areas, markers, error bars, correlation ribbon; legend **`fit`** |
| **`scatterchart`** | Points + optional regression/correlation |
| **`radarchart`** | Radar / spider series |
| **`polarareachart`** | Polar area slices |
| **`combochart`** | Shares bar + line drawing; **`ComboChartOpacity`**; dual **Y** |
| **`comparisonchart`** | **`ComparisonLayout`**, **`ComparisonChartConfig`** per panel + optional **`generalTitle`** |

**Supporting modules** (used internally; not all re-exported on **`Charts`**): **`chartPadding.ts`**, **`legendTextLayout.ts`**, **`cartesianLegendLayout.ts`**, **`axisTitleLayout.ts`**, **`segmentValueLabel.ts`**.

## Appearance & backgrounds

**`options.appearance` (`ChartAppearanceExtended`)** mirrors **`CanvasConfig`**-style backgrounds so charts can match scene canvases:

1. Base **`customBg`** / **`backgroundImage`** / **`backgroundGradient`** / **`backgroundColor`**
2. **`bgLayers`** (**`drawBackgroundLayers`**)
3. **`patternBg`** (**`EnhancedPatternRenderer`**)
4. **`noiseBg`**

Comparison panels can override **`appearance`** per cell where the merger allows; combo and single charts pass **`appearance`** at the top level.

**`Charts.paintChartCanvasBackground`** — same stack for custom **`SKRSContext2D`** drawing next to Apexify charts.

## Choosing façade vs `Charts` namespace

| Need | Call |
|------|------|
| Typed **`createChart`** on your painter | **`painter.createChart(...)`** |
| Direct calls without **`ApexPainter`** | **`Charts.createLineChart`** (etc.) from **`import { Charts } from 'apexify.js'`** |
| Dual-axis bar + line | **`painter.createComboChart`** or **`Charts.createComboChart`** |
| Two charts in one image | **`painter.createComparisonChart`** or **`Charts.createComparisonChart`** |

Individual **`create*Chart`** functions are **not** flat-exported from the package root — use the **`Charts`** namespace.

**Unified tooling types:** **`ChartType`**, **`ChartData`**, **`ChartOptions`** (discriminated unions) are exported from **`Charts`** for wrappers and generators.
