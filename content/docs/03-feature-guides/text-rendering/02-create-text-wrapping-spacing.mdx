# Wrapping, line breaks & spacing {#02-create-text-wrapping-spacing}

Back: [Text overview](/docs#00-create-text-overview)

Branching inside **`EnhancedTextRenderer.renderText`**:

1. If **`textOnCurve`** → curved renderer (**ignores** **`maxWidth`** wrapping — use **`\\n`** only for stacked arcs).
2. Else if **`maxWidth`** → word-wrap pipeline.
3. Else **`renderSingleLine`** — still splits on **`\\n`** for manual multi-line blocks.

---

## Word wrap (`maxWidth`)

- Splits on **spaces** per word; **`measureText`** decides overflow.
- Hard **`\\n`** in **`text`** starts a new explicit segment — blank lines preserved when multiple **`\\n`** segments exist (same trimming rules as **`renderWrappedText`**).
- When adding the next word would exceed **`maxWidth`**, current line commits and the word starts a fresh line.

---

## Height cap (`maxHeight`)

**`maxLines = floor(maxHeight / lineHeight)`** where **`lineHeight`** on **`TextProperties`** defaults to multiplier **`1.4`** times **`fontSize`** when omitted.

When **`maxLines`** is exceeded mid-wrap, remaining content collapses to **`'...'`** on that line (same behaviour in **`TextMetricsCreator.wrapText`**).

---

## Letter & word spacing

| Field | Canvas mapping |
|-------|----------------|
| **`letterSpacing`** | **`ctx.letterSpacing = '${n}px'`** |
| **`wordSpacing`** | **`ctx.wordSpacing = '${n}px'`** |

Included automatically in **`measureText`** widths — affects curved layout too.

---

## Examples

**Manual lines (no auto wrap)**

```typescript
await painter.createText(
  {
    text: 'Line one\nLine two',
    x: 40,
    y: 60,
    font: { size: 28, family: 'Arial' },
    lineHeight: 1.35,
  },
  buffer
);
```

**Auto wrap + tall box clamp**

```typescript
await painter.createText(
  {
    text: longParagraph,
    x: 40,
    y: 80,
    maxWidth: 520,
    maxHeight: 220,
    lineHeight: 1.45,
    letterSpacing: 0.5,
    wordSpacing: 2,
  },
  buffer
);
```
