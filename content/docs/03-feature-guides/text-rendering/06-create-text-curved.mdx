# Text on a circular arc (`textOnCurve`) {#06-create-text-curved}

Back: [Text overview](/docs#00-create-text-overview)

```typescript
textOnCurve?: {
  sweepAngle: number;   // degrees — total arc covered by full string
  radius?: number;      // px — optional; defaults so arc length ≈ measured width
  up?: boolean;         // default true = ∩ smile; false = ∪ frown
};
```

---

## Anchor semantics

**`(x, y)`** is the **centre of the string**: the middle glyph sits at the arc midpoint (`EnhancedTextRenderer.renderCurvedLine`).

- **`up !== false`**: centre **`cy = y + R`** (arc bulges upward).
- **`up === false`**: **`cy = y - R`** (bulge downward).

---

## Geometry

Let **`θ = sweepAngle * π / 180`**, **`W = measureText(line).width`**.

Default radius **`R = W / θ`** so arc length **`R·θ`** matches measured width (when **`radius`** omitted).

Each character:

1. Computes cumulative substring widths (**left**, **right**) so **kerning-aware** centres beat naive fixed pitch (`chars.join` measurement — comment in source warns per-char sums ignore kerning).
2. Maps arc length **`centerDist`** → angle **`α`** along circle.
3. Positions glyph at **`(px, py)`**, rotates tangent via **`renderRotatedGlyph`**.

---

## Sweep limits

If **`sweepAngle <= 0`**, **`>= 360`**, or line empty → falls back to **straight** **`renderTextLine`**.

---

## Newlines

**`text.split('\n')`** — each row becomes its **own** arc with **`y += lineHeight * index`**.

**`maxWidth`** does **not** word-wrap in curve mode — keep lines short or insert **`\\n`** yourself.

---

```typescript
await painter.createText(
  {
    text: 'APEXIFY.JS',
    x: 400,
    y: 280,
    font: { size: 44, family: 'Arial' },
    textAlign: 'center',
    textOnCurve: { sweepAngle: 68, up: true },
    gradient: {
      type: 'linear',
      colors: [
        { stop: 0, color: '#fef3c7' },
        { stop: 1, color: '#fb923c' },
      ],
    },
    stroke: { color: '#451a03', width: 2 },
  },
  buffer
);
```
