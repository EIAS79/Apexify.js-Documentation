# Text rendering (`createText`) — overview {#00-create-text-overview}

**`await painter.createText(textPropsOrArray, canvasBuffer)`** draws styled Unicode strings onto an existing PNG **`canvasBuffer`** (from **`createCanvas`**, **`createImage`**, etc.). Each draw returns a **new** **`Buffer`**.

Back: [Feature guides hub](/docs#feature-guides-hub)

---

## What you pass

| Argument | Role |
|----------|------|
| **`textProps`** | One **`TextProperties`** **or** **array** (processed **top → bottom**). |
| **`canvasBuffer`** | **`Buffer`** or **`CanvasResults`** with **`buffer`**. |

**Required on each object:** **`text`**, **`x`**, **`y`**.

---

## Renderer stack

**`TextCreator`** (`lib/Canvas/extended/TextCreator.ts`) is thin: it loads the bitmap into a fresh canvas and calls **`EnhancedTextRenderer.renderText`** (`lib/Canvas/utils/Texts/enhancedTextRenderer.ts`) for every entry.

Legacy **`drawText`** / **`WrappedText`** in **`lib/Canvas/utils/Texts/textProperties.ts`** power **`TextObject`** helpers elsewhere — **`createText` does not call them**.
