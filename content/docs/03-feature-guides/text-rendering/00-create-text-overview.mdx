# Text rendering (`createText`) — overview {#00-create-text-overview}

**`await painter.createText(textPropsOrArray, canvasBuffer)`** draws styled Unicode strings onto an existing PNG **`canvasBuffer`** (from **`createCanvas`**, **`createImage`**, etc.). Each call returns a **new** **`Buffer`**.

Back: [Feature guides hub](/docs#feature-guides-hub)

---

## What you pass

| Argument | Role |
|----------|------|
| **`textProps`** | One **`TextProperties`** **or** **array** (processed **in order**, top → bottom on the canvas). |
| **`canvasBuffer`** | **`Buffer`** or **`CanvasResults`** with **`buffer`**. |

**Required on each object:** **`text`**, **`x`**, **`y`**.

---

## `TextProperties`: nested vs flat

Types live in **`lib-next/types/text.ts`** (exported from **`apexify.js`** / **`apexify.js/types`**).

| Prefer (grouped) | Replaces legacy top-level |
|--------------------|---------------------------|
| **`font.{ size, family, name, path }`** | **`fontSize`**, **`fontFamily`**, **`fontName`**, **`fontPath`** |
| **`decorations.{ underline, overline, strikethrough, bold, italic }`** | flat **`underline`**, **`bold`**, … |
| **`effects.{ highlight, glow, shadow }`** | flat **`highlight`**, **`glow`**, **`shadow`** |
| **`layout.{ lineHeight, letterSpacing, wordSpacing, maxWidth, maxHeight }`** | same names at root |
| **`placement.{ textAlign, textBaseline, rotation }`** | flat **`textAlign`**, **`rotation`**, … |
| **`fill.{ color, gradient, opacity }`** | flat **`color`**, **`gradient`**, **`opacity`** |

If both nested and flat are set, **nested wins** (see **`resolveText*`** helpers in **`text.ts`**). New examples in the **[field matrix](/docs#09-create-text-properties-matrix)** use the grouped style; older guides still show flat keys — both work.

---

## Renderer stack

**`TextCreator`** (`lib-next/text/text-creator.ts`) loads the bitmap, draws it onto a working canvas, and calls **`EnhancedTextRenderer.renderText`** (`lib-next/text/enhanced-text-renderer.ts`) for each entry.

**`measureText`** uses **`TextMetricsCreator`** with the same **`TextProperties`** shape and **`resolveText*`** rules where applicable.
