# Video & FFmpeg (`createVideo`) — overview {#00-video-overview}

**`await painter.createVideo(options[, painterOpts])`** routes FFmpeg/ffprobe-backed work: probe clips, extract frames (single, timed lists, intervals, full dumps), thumbnails and previews, transcode and compress, trim and timeline edits, geometry (`rotate`, `crop`), audio extraction and mixes, filters and colour (`applyEffects`, `colorCorrect`, `applyLUT`), watermarks and fades, raster text (`addText`, `addAnimatedText`), compositing (`merge`, `splitScreen`, `pictureInPicture`, `createFromFrames`, `createLoop`), scene detection, stabilization, transitions, batched pipelines, optional **`onProgress`** on branches that forward it (see **[Progress](/docs#11-on-progress)**).

**Requirements:** **`ffmpeg`** and **`ffprobe`** on **`PATH`**; missing binaries throw an install hint. **`Buffer`** and **`http(s)`** sources are staged through temp files where FFmpeg needs paths (**more:** [Imperative helpers](/docs#08-imperative-video-helpers)).

**Implementation:** **`VideoCreator.createVideo`** in **`lib-next/video/video-creator.ts`**, wired via **`VideoStack`** (**`lib-next/video/video-stack.ts`**) into **`VideoHelpers`** (**`lib-next/video/video-helpers.ts`**).

---

## Signature & asset refs

```typescript
await painter.createVideo(options: VideoCreationOptions, painterOpts?: PainterAssetRefsOptions);
```

When **`painterOpts.resolveAssetRefs`** is **`true`**, **`options`** is passed through **`maybeResolveRefs`** (**`$…`** placeholders, same pattern as **`createImage`** and **`createGIF`** on **`ApexPainter`**, **`lib-next/apex-painter/main.ts`**). To resolve refs yourself with the flag off, use **`prepareForRender`**.

---

## One operation per call (router)

Design for **exactly one** operation key in **`options`** besides **`source`** (and **`onProgress`** when used). **`VideoCreator.createVideo`** is a linear **`if`** chain: **the first truthy branch wins**; later keys are ignored. For example **`getInfo`** is checked **before** **`extractFrame`**, **`convert`**, **`trim`**, **`merge`**, **`createFromFrames`**, **`detectFormat`**, **`exportPreset`**, **`addAnimatedText`**, and so on—**`detectFormat`** is **late** in the chain, **after** **`createFromFrames`**. Full order is defined in **`lib-next/video/video-creator.ts`**. The **[options reference card](/docs#07-video-creation-options-matrix)** maps each key to its deep-dive page.

Some operations ignore top-level **`source`** (they use their own paths or **`videos`** arrays)—the type still requires **`source`**; tutorials call that out (**`merge`**, **`splitScreen`**, **`batch`**, …).

## Scenes → video

**`painter.renderSceneToVideoFrames(scene, video)`** renders the scene to PNG, merges optional frame slots, and delegates to **`createVideo`**. **`video.options.createFromFrames`** is **required** (**`lib-next/scene/render-scene-to-video.ts`**). **`video.sceneRender.resolveAssetRefs`** defaults **`true`** — **`scene`** asset refs use **`resolveSceneRenderInputAssets`** unless you opt out (**`main.ts`**).
