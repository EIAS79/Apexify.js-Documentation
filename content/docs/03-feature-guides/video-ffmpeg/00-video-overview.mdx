# Video & FFmpeg (`createVideo`) — overview {#00-video-overview}

**`await painter.createVideo(options[, painterOpts])`** is the main entry for FFmpeg/ffprobe work: probe clips, extract frames, transcode, trim, audio mixes, **`addTextOverlay`**, compositing, and more.

**Multi-step editor workflows** (trim → splice → captions → audio in one stack) live under **[Advanced → Video pipeline](/docs#01-video-pipeline)** — not in this feature guide.

**Requirements:** **`ffmpeg`** and **`ffprobe`** on **`PATH`**. **`Buffer`** and **`http(s)`** sources are staged to temp files when required (**more:** [Imperative helpers](/docs#08-imperative-video-helpers)).

**Low-level stack:** **`painter.video`** — **`getVideoInfo`**, frame extract helpers, and **`videoPipeline()`** ([advanced](/docs#00-video-advanced-hub)).

---

## Signature & asset refs

```typescript
await painter.createVideo(options: VideoCreationOptions, painterOpts?: PainterAssetRefsOptions);
```

When **`painterOpts.resolveAssetRefs`** is **`true`**, **`options`** is passed through **`maybeResolveRefs`** (**`$…`** placeholders, same pattern as **`createImage`** and **`createGIF`** on **`ApexPainter`**, **`lib-next/apex-painter/main.ts`**). To resolve refs yourself with the flag off, use **`prepareForRender`**.

---

## One operation per call (router)

Design for **exactly one** operation key in **`options`** besides **`source`** (and **`onProgress`** when used). **`VideoCreator.createVideo`** is a linear **`if`** chain: **the first truthy branch wins**; later keys are ignored. For example **`getInfo`** is checked **before** **`extractFrame`**, **`convert`**, **`trim`**, **`merge`**, **`createFromFrames`**, **`detectFormat`**, **`exportPreset`**, **`addTextOverlay`**, and so on—**`detectFormat`** is **late** in the chain, **after** **`createFromFrames`**. The **[options reference card](/docs#07-video-creation-options-matrix)** maps each key to its deep-dive page.

Some operations ignore top-level **`source`** (they use their own paths or **`videos`** arrays)—the type still requires **`source`**; tutorials call that out (**`merge`**, **`splitScreen`**, **`batch`**, …).

## Scenes → video

**`painter.renderSceneToVideoFrames(scene, video)`** renders the scene to PNG, merges optional frame slots, and delegates to **`createVideo`**. **`video.options.createFromFrames`** is **required** (**`lib-next/scene/render-scene-to-video.ts`**). **`video.sceneRender.resolveAssetRefs`** defaults **`true`** — **`scene`** asset refs use **`resolveSceneRenderInputAssets`** unless you opt out (**`main.ts`**).
