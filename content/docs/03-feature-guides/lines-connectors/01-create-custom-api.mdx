# `path2d.custom` — custom lines, segments & validation {#01-create-custom-api}

Back: [Lines overview](/docs#00-custom-lines-overview)

On **`ApexPainter`**, connector lines are drawn with **`painter.path2d.custom`** (**`(options, buffer)`** order). Internally this runs **`runDrawCustomLines`** (**`lib-next/apex-painter/path-custom-lines.ts`**) → **`customLines`** (**`lib-next/path/custom-lines.ts`**).

```typescript
const buffer = await painter.path2d.custom(
  customOptionsObjectOrArray,
  canvasBuffer /* Buffer | CanvasResults */
);
```

Loads **`canvasBuffer`** onto a working canvas, renders **`customLines(ctx, opts)`**, returns a new **PNG** **`Buffer`**.

---

## Required fields (every entry)

| Field | Rule |
|-------|------|
| **`startCoordinates`** | **`{ x, y }`** numbers |
| **`endCoordinates`** | **`{ x, y }`** numbers |

Validation is strict — missing **`x`/`y`** throws (**`validateCustomOptions`** in **`path-custom-lines.ts`**).

---

## Batch processing

Pass an **array** of **`CustomOptions`** — segments are processed **in order**. Later segments can **chain** when **`lineStyle.singleLine`** is **`true`** (see below).

---

## **`singleLine`** chains

When **`lineStyle.singleLine`** is set:

- First segment: **`moveTo(start)`**; later segments reuse **`previousEndCoordinates`** as implicit **`start`** so **polylines** share one context sub-path until **`singleLine`** ends.
- Leaving **`singleLine`** (next item without it) triggers **`stroke`** + optional **`applyStroke`** cleanup.

Use this for **multi-segment** connectors without lifting the pen.

---

## Minimal example

```typescript
const base = await painter.createCanvas({ width: 400, height: 300, colorBg: '#f8fafc' });

const withLine = await painter.path2d.custom(
  {
    startCoordinates: { x: 40, y: 260 },
    endCoordinates: { x: 360, y: 40 },
    lineStyle: { width: 4, color: '#2563eb', lineCap: 'round' },
  },
  base.buffer
);
```

## Naming note

Older docs and snippets refer to **`createCustom`**; in **`lib-next`** the same capability lives on **`painter.path2d.custom`** so vector **`create`/`draw`** and connector **`custom`** stay grouped.
