# Hit testing — paths & regions {#05-hit-testing-paths}

Back: [Lines overview](/docs#00-custom-lines-overview)

Vector **lines** chapters focus on **`painter.path2d`** and **`painter.detect`** working from the same **`PathCommand[]`** / **`Path2D`** shapes: whatever you **draw** can be **tested** with **`await painter.detect.path(…)`**.

---

## API entry point

Use **`painter.detect`** (not legacy top-level **`isPointInPath`** names on **`ApexPainter`**):

| Method | Use when |
|--------|----------|
| **`detect.path(path, x, y, options?)`** | Arbitrary **`Path2D`** or **`PathCommand[]`** |
| **`detect.region(region, x, y, options?)`** | **`HitRegion`** primitives (**`rect`**, **`circle`**, **`ellipse`**, **`polygon`**, **`path`**, **`custom`**) |
| **`detect.anyRegion(regions, x, y, options?)`** | UI layers — first hit wins; read **`hitRegion`** |
| **`detect.distance(region, x, y)`** | Proximity to **`rect` / `circle` / `ellipse` / `polygon`** |

```typescript
const path = painter.path2d.create([
  { type: "roundedRect", x: 20, y: 40, width: 200, height: 80, radius: 12 },
]);

const rendered = await painter.path2d.draw(canvasBuf, path, {
  stroke: { color: "#e2e8f0", width: 2 },
  fill: { color: "#334155", opacity: 0.9 },
});

const pick = await painter.detect.path(path, 120, 80, { fillRule: "nonzero" });
```
