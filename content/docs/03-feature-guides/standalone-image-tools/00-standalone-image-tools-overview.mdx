# Standalone image tools {#00-standalone-image-tools-overview}

Back: [Feature guides hub](/docs#feature-guides-hub)

These helpers operate on **whole images** (filesystem paths, URLs, or **`Buffer`s**) using **Sharp** and shared **`lib-next/core/general-functions.ts`** / **`lib-next/output/*`** pipelines. They are **not** the same as compositing with **`createImage`** on a canvas — use them when you want **resize**, **polygon crop**, **dominant-colour analysis**, **chroma-style removal**, **multi-layer blend**, **stitching**, **re-encode**, etc. **outside** a scene graph.

On **`lib-next`** **`ApexPainter`**, everything below lives on the **`painter.image`** facet (**`PainterImageUtils`**), lazily the same object as **`painterImageUtils`** in **`lib-next/image/painter-image-utils.ts`**. There are **no** top-level **`painter.resize`** / **`painter.cropImage`** methods on the class.

## What lives on `painter.image` (summary)

| Area | Methods |
|------|---------|
| Layout & encode | **`stitchImages`**, **`createCollage`**, **`compress`**, **`extractPalette`**, **`validHex`** (sync) |
| Resize & format | **`resize`**, **`imgConverter`** |
| Filters | **`effects`** (**`ImageFilter[]`**) |
| Colour | **`colorsFilter`**, **`colorAnalysis`**, **`colorsRemover`** |
| Remote BG | **`removeBackground`** |
| Composite | **`blend`**, **`masking`**, **`gradientBlend`** |
| Polygon crop | **`cropImage`** |

**Overlap with [Raster utilities](/docs#03-batch-stitch-compress-output):** stitch, collage, compress, and palette are documented there too as part of **`batch`/`chain`** workflows; this track is the **full method reference** for **`painter.image`**.
