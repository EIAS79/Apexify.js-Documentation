# Batch, stitch, compression & output encoding {#03-batch-stitch-compress-output}

Back: [Raster overview](/docs#00-advanced-raster-overview)

**Disk files:** **`painter.save`**, **`saveMultiple`**, and **`OutputFormat`** are documented under **[Batch, save & output](/docs#00-batch-save-output-overview)**.

This page covers **`painter.batch`**, **`painter.chain`**, the **`painter.image`** utility bundle (Sharp-backed layout and compression), and the two ways to turn a **PNG** **`Buffer`** into other representations: constructor-driven **`painter.outPut`** vs explicit **`painter.output.*`** helpers.

---

## `batch(operations: BatchOperation[]): Promise<Buffer[]>`

**`lib-next/batch/batch-operations.ts`** — **`Promise.all`** over one job per operation (parallel).

| **`type`** | Behaviour |
|------------|-----------|
| **`canvas`** | **`createCanvas(config)`** → take **`.buffer`** |
| **`image`** | Temporary **`800×600`** canvas, then **`createImage(config, base)`** |
| **`text`** | Same base pattern with **`createText(config, base)`** |

```typescript
const [a, b] = await painter.batch([
  { type: "canvas", config: { width: 120, height: 40, color: "#0f172a" } },
  { type: "text", config: { text: "B", x: 60, y: 24, font: { size: 22, family: "Arial" } } },
]);
```

## `chain(operations: ChainOperation[]): Promise<Buffer>`

Sequential pipeline. Each step:

```typescript
{ method: string; args: unknown[] }
```

**`method`** names a function on **`painter`**. **Dotted paths** resolve nested facets (e.g. **`path2d.draw`**, **`pixels.setColor`**) via **`lib-next/batch/batch-operations.ts`**.

**Pipe prior output:** use the string **`'current'`** or **`{ __isCurrentBuffer: true }`** in **`args`** where the next step should receive the **previous **`Buffer`**.

Each step must return a **`Buffer`** or **`{ buffer }`** (e.g. **`createCanvas`**).

```typescript
const out = await painter.chain([
  { method: "createCanvas", args: [{ width: 400, height: 200, color: "#111827" }] },
  {
    method: "path2d.draw",
    args: [
      "current",
      [
        { type: "moveTo", x: 40, y: 100 },
        { type: "lineTo", x: 360, y: 100 },
      ],
      { stroke: { color: "#f472b6", width: 3 } },
    ],
  },
]);
```

## `painter.image` — stitch, collage, compress, palette, `validHex`

Implemented in **`lib-next/image/painter-image-utils.ts`** (Sharp + shared **`lib-next/output/*`** helpers). These methods do **not** require a prior **`createCanvas`** unless you compose them in **`chain`**.

**Full method list** (resize, crop, **`effects`**, **`colorsRemover`**, **`blend`**, **`masking`**, …): **[Standalone image methods](/docs#01-standalone-image-methods)**.

### `stitchImages(images, options?)`

**`images`**: **`string`** (URL or path relative to **`process.cwd()`**) or **`Buffer`**.

**`StitchOptions`** (see **`lib-next/types/batch.ts`**): **`direction`** (**`horizontal`** default, **`vertical`**, **`grid`**), **`overlap`**, **`spacing`**, **`blend`**, etc.

```typescript
const strip = await painter.image.stitchImages(["./a.png", "./b.png"], {
  direction: "horizontal",
  spacing: 4,
});
```

### `createCollage(images, layout)`

**`CollageLayout`**: **`type`** (**`grid`**, **`masonry`**, **`carousel`**, **`custom`**), **`columns`**, **`rows`**, **`spacing`**, **`background`**, **`borderRadius`**, …

### `compress(image, options?)`

**`CompressionOptions`**: **`quality`**, **`format`** (**`jpeg`**, **`webp`**, **`avif`**, …), optional **`maxWidth`** / **`maxHeight`**, **`progressive`** (JPEG).

```typescript
const webp = await painter.image.compress(buf, { format: "webp", quality: 82 });
```

### `extractPalette(image, options?)`

Dominant colours — **`PaletteOptions`**: **`count`**, **`method`**, **`format`**.

### `painter.image.validHex(hexColor: string): boolean`

Validates **`/^#[0-9a-fA-F]{6}$/`** (throws on mismatch per **`lib-next/core/color.ts`** usage in **`painterImageUtils`**). Also documented under [Encoded output](/docs#02-output-format-and-outPut).

## Two output surfaces

### 1. **`painter.outPut(buffer)`** — constructor **`type`**

**`new ApexPainter({ type: 'base64' })`** then **`await painter.outPut(buf)`** returns whatever **`lib-next/output/buffer-output.ts`** maps for that **`type`** (**`buffer`**, **`dataURL`**, **`blob`**, **`base64`**, **`arraybuffer`**, **`url`**, …).

### 2. **`painter.output`** — explicit encoders

**`PainterOutput`** (**`public-types.ts`**): **`dataURL`**, **`base64`**, **`blob`**, **`arrayBuffer`** — pure transforms, **ignore** constructor **`type`**. **`output.url`** uploads a PNG to Imgur (env credentials required).

```typescript
const dataUrl = painter.output.dataURL(buf);
const b64 = painter.output.base64(buf);
```

Use **`output`** when you want **multiple encodings** from one painter instance without re-instantiating for each **`type`**.

## Importing utilities without `ApexPainter`

Low-level Sharp / conversion helpers still live under **`lib-next/core/general-functions.ts`**, **`lib-next/output/*`**, etc. **`painter.image`** is the supported **ergonomic** surface for apps already holding an **`ApexPainter`**.
