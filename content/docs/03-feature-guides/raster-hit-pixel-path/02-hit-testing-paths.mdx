# Hit testing — paths & regions {#02-hit-testing-paths}

Back: [Raster overview](/docs#00-advanced-raster-overview) · [Path2D · `draw`](/docs#04-path2d-draw-path)

Hit testing is grouped on **`painter.detect`** (**`PainterHitDetect`** in **`lib-next/apex-painter/public-types.ts`**), backed by **`HitDetectionCreator`** (**`lib-next/pixels/hit-detection-creator.ts`**).

Methods are typed as **`async`** so they match the rest of the painter surface; the underlying **`HitDetectionCreator`** work is **synchronous** (a **`Promise`** resolves on the next microtask with the same result).

---

## `painter.detect` methods

| Method | Purpose |
|--------|---------|
| **`path(path, x, y, options?)`** | **`Path2D`** or **`PathCommand[]`** — fill (and optional stroke) test at **`(x,y)`** in **canvas space** |
| **`region(region, x, y, options?)`** | One **`HitRegion`** discriminated union |
| **`anyRegion(regions, x, y, options?)`** | First matching region wins; result may include **`hitRegion`** index |
| **`distance(region, x, y)`** | Numeric distance helper for **`rect`**, **`circle`**, **`ellipse`**, **`polygon`** (see below) |

```typescript
import type { PathCommand } from "apexify.js"; // or your types import path

const cmds: PathCommand[] = [
  { type: "moveTo", x: 0, y: 0 },
  { type: "lineTo", x: 100, y: 0 },
  { type: "lineTo", x: 50, y: 80 },
  { type: "closePath" },
];

const { hit, hitType } = await painter.detect.path(cmds, 50, 40, { fillRule: "nonzero" });

const zone = await painter.detect.region(
  { type: "rect", x: 10, y: 10, width: 120, height: 48 },
  64,
  32
);

const multi = await painter.detect.anyRegion(
  [
    { type: "circle", x: 100, y: 100, radius: 30 },
    { type: "rect", x: 200, y: 50, width: 40, height: 40 },
  ],
  105,
  98
);
if (multi.hit && multi.hitRegion !== undefined) {
  // index into the regions array
}

const d = await painter.detect.distance({ type: "circle", x: 0, y: 0, radius: 50 }, 80, 0);
```

## `HitDetectionResult`

| Field | Meaning |
|-------|---------|
| **`hit`** | **`true`** if the point matches |
| **`hitType`** | **`'fill'`**, **`'stroke'`**, or **`'outside'`** |
| **`distance`** | For **`circle`** fill hit, centre distance; often **`0`** or **`undefined`** when not meaningful |
| **`hitRegion`** | Set by **`anyRegion`** — **number** index into the **`regions`** array |

---

## `HitDetectionOptions`

| Option | Role |
|--------|------|
| **`includeStroke`** | With **`strokeWidth`**, enables stroke band logic where implemented |
| **`strokeWidth`** | Used for **rect** / **circle** stroke rings and **`path`** stroke branch |
| **`tolerance`** | Extra slack on **rect** stroke detection (default **`0`**) |
| **`fillRule`** | **`'nonzero'`** / **`'evenodd'`** for **`path`** tests and **`{ type: 'path', … }`** regions |

The **`region`** field on **`HitDetectionOptions`** is **not** used by **`detect.path`** — pass geometry as the first argument or embed a **`{ type: 'path', path }`** **`HitRegion`**.

---

## `detect.path` — stroke caveat

Stroke testing is gated on **`includeStroke`** and a truthy **`strokeWidth`**. The implementation calls **`ctx.isPointInStroke(path2D, x, y)`** without mirroring your real-world **`lineWidth`** from **`drawPath`** into that scratch context, so **stroke hits follow the engine default width** unless your visual stroke matches that behaviour. For precise “fat line” picking, prefer **fill**-expanded hit geometry or test a parallel **`PathCommand[]`** outline.

## `HitRegion` union

| **`type`** | Behaviour |
|------------|-----------|
| **`rect`** | Axis-aligned **`x,y,width,height`**. With **`includeStroke`** + **`strokeWidth`**, **`hitType`** can be **`stroke`** vs **`fill`** |
| **`circle`** | Centre **`x,y`**, **`radius`**; optional annulus when stroke options set |
| **`ellipse`** | **`radiusX`**, **`radiusY`**, optional **`rotation`** (radians). Fill test; **`includeStroke`** does not refine **`hitType`** |
| **`polygon`** | **`points[]`** (≥ **3**), ray casting |
| **`path`** | Delegates to **`detect.path`** with **`region.path`** and optional **`fillRule`** |
| **`custom`** | **`check(x, y) => boolean`** |

---

## `getDistanceToRegion`

| **`type`** | Metric |
|------------|--------|
| **`rect`** | Distance to axis-aligned box (outside **0** when inside) |
| **`circle`** | **`abs(distanceToCentre − radius)`** |
| **`ellipse`** | Approximate distance to ellipse boundary |
| **`polygon`** | Minimum distance to polygon edges |
| **`path`**, **`custom`** | Returns **`0`** (no analytic distance) |

Useful for hover “glow” or snap UX for primitives — not for arbitrary **`Path2D`** distance fields.
