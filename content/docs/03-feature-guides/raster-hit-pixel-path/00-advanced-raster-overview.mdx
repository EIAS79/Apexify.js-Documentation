# Raster, pixels & hit testing {#00-advanced-raster-overview}

Back: [Feature guides hub](/docs#feature-guides-hub)

In **`lib-next`**, **`ApexPainter`** exposes raster work through **small grouped surfaces** (facades) instead of a flat list of methods on the class root. That keeps **pixels**, **vector paths**, **hit testing**, and **encode / upload helpers** discoverable without crowding **`createCanvas` / `createImage` / `createText`**.

---

## Facades at a glance

| Facet | Property | Typical use |
|-------|----------|-------------|
| **Pixels** | **`painter.pixels`** | **`ImageData`**-style **`Uint8ClampedArray`**, CPU filters, single-pixel reads/writes |
| **Path2D** | **`painter.path2d`** | Build **`PathCommand[]`** → **`Path2D`**, **`draw`** stroke/fill onto an existing PNG, or **`custom`** connector lines |
| **Hit test** | **`painter.detect`** | **`isPointInPath`** / **`isPointInRegion`** equivalents for UI hit targets |
| **Image utils** | **`painter.image`** | **`stitchImages`**, **`createCollage`**, **`compress`**, **`extractPalette`**, **`validHex`**, resize, blend, … |
| **Structured output** | **`painter.output`** | **`dataURL`**, **`base64`**, **`blob`**, **`arrayBuffer`**, **`url`** (Imgur) from a finished **PNG** buffer |
| **Format `outPut`** | **`painter.outPut(buf)`** | Still driven by **`new ApexPainter({ type: '…' })`** — same idea as before: one buffer → string/Blob/ArrayBuffer matching constructor **`type`** |

**Orchestration** (unchanged on the class): **`painter.batch`**, **`painter.chain`** — see [Batch & utilities](/docs#03-batch-stitch-compress-output).

---

## When to use this track

| Goal | Lead APIs |
|------|-----------|
| Read or write RGBA buffers | **`await painter.pixels.getData`**, **`setData`**, **`getColor`**, **`setColor`** |
| Per-pixel or built-in CPU filters on a region | **`await painter.pixels.manipulate`** |
| Build geometry then stroke/fill on an existing raster | **`painter.path2d.create`**, **`await painter.path2d.draw`** — command table in **[Path2D](/docs#04-path2d-draw-path)** |
| Decorative connectors (smooth / bezier / markers) on top of a buffer | **`await painter.path2d.custom`** (**`CustomOptions`**) — see [Custom lines](/docs#03-custom-lines-arrows-markers-textures) |
| Click maps, hover zones, distance-to-edge | **`await painter.detect.path`**, **`region`**, **`anyRegion`**, **`distance`** |
| Parallel canvas/image/text jobs, pipelines, layout, compression | **`painter.batch`**, **`painter.chain`**, **`painter.image.*`** |

## Types (source of truth)

- **`PixelData`**, **`PixelManipulationOptions`** — **`lib-next/types/pixels.ts`**
- **`HitRegion`**, **`HitDetectionOptions`**, **`HitDetectionResult`** — **`lib-next/types/hit-detection.ts`**
- **`PathCommand`** — **`lib-next/types/pathCommands.ts`** (built via **`lib-next/foundation/path-cmd.ts`**)
- **`Path2DDrawOptions`** — **`lib-next/types/path2d-draw.ts`**
- **`CustomOptions`** — **`lib-next/types/path.ts`**
- **`BatchOperation`**, **`ChainOperation`**, **`StitchOptions`**, **`CollageLayout`**, **`CompressionOptions`**, **`PaletteOptions`** — **`lib-next/types/batch.ts`**

## Minimal usage sketch

```typescript
import { ApexPainter } from "apexify.js";

const painter = new ApexPainter({ type: "buffer" });

const { buffer } = await painter.createCanvas({ width: 400, height: 200, color: "#1e293b" });

const path = painter.path2d.create([
  { type: "moveTo", x: 40, y: 100 },
  { type: "lineTo", x: 360, y: 100 },
]);

const drawn = await painter.path2d.draw(buffer, path, {
  stroke: { color: "#38bdf8", width: 4, lineCap: "round" },
});

const { hit, hitType } = await painter.detect.path(path, 200, 100, { fillRule: "nonzero" });

const px = await painter.pixels.getColor(drawn, 200, 100);
```
