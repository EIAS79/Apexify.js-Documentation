# Raster, pixels & hit testing {#00-advanced-raster-overview}

Back: [Feature guides hub](/docs#feature-guides-hub)

These **`ApexPainter`** APIs sit beside the main **canvas / image / text / GIF / lines / charts / video** tracks: raw **RGBA** access, **vector paths** rasterised with **`drawPath`**, **hit testing**, and **batch / IO helpers** wired through **`lib/Canvas/utils/general`**.

---

## When to use this track

| Goal | Lead APIs |
|------|-----------|
| Read or write **`ImageData`**-style buffers | **`getPixelData`**, **`setPixelData`**, **`getPixelColor`**, **`setPixelColor`** |
| Per-pixel or built-in CPU filters on a region | **`manipulatePixels`** |
| Build geometry then stroke/fill on an existing PNG | **`createPath2D`**, **`drawPath`** — full command table in **[Path2D](/docs#04-path2d-draw-path)** |
| Click maps, hover zones, distance-to-edge | **`isPointInPath`**, **`isPointInRegion`**, **`isPointInAnyRegion`**, **`getDistanceToRegion`** |
| Parallel canvas/image/text renders, pipelines, stitching, compression | **`batch`**, **`chain`**, **`stitchImages`**, **`createCollage`**, **`compress`**, **`extractPalette`**, **`outPut`** |

Implementation anchors:

- **`PixelDataCreator`** — **`lib/Canvas/extended/PixelDataCreator.ts`**
- **`Path2DCreator`** — **`lib/Canvas/extended/Path2DCreator.ts`**
- **`HitDetectionCreator`** — **`lib/Canvas/extended/HitDetectionCreator.ts`**
- **`batchOperations`**, **`chainOperations`**, **`stitchImages`**, **`createCollage`**, **`compressImage`**, **`extractPalette`** — **`lib/Canvas/utils/general/`**
