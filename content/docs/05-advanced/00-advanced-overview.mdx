---

## Composition model

Most flows are:

1. **`createCanvas`** → base bitmap (+ backgrounds).  
2. **`createImage`** / **`createText`** / **`createCustom`** / **`drawPath`** → layers on that buffer.  
3. **`save`** / **`outPut`** → files or encoded strings — see [Raster · batch & utilities](/docs#03-batch-stitch-compress-output).

Charts and GIF/video helpers either return buffers or orchestrate their own pipelines; each track documents ordering.

## Engines you touch indirectly

| Engine | Role |
|--------|------|
| **`@napi-rs/canvas`** | Raster canvas, paths, text rendering |
| **Sharp** | Image filters, compression, palette extraction, many URL/path loads |
| **FFmpeg / ffprobe** | Optional; video discovery, transcode, compose — [Video](/docs#00-video-overview) |

---

## Performance intuition

- **`createImage`** filters often use **Sharp** (good for full-frame work).  
- **`getPixelData`** / **`manipulatePixels`** are **CPU loops** — best for small regions or custom logic ([Raster](/docs#00-advanced-raster-overview)).  
- **`batch`** runs independent jobs in **parallel** (`Promise.all`); **`chain`** runs **sequentially** and pipes buffers — [utilities](/docs#03-batch-stitch-compress-output).
