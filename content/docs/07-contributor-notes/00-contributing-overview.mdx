## Documentation style

| Rule | Why |
|------|-----|
| **Stable anchors** | Section headings use **`{#anchor-id}`** so links don’t rot |
| **`00-start-here` / `01-beginner-guide`** | Set **`{#basename}`** to match **`*.mdx`** filename (**without extension**) — links use **`/docs#00-start-here`**, **`/docs#01-install-and-setup`**, … |
| **Small pages** | Prefer several **`*.mdx`** files over one huge scroll |
| **Hub + chapters** | Overview (`00-*-overview`) lists numbered follow-ups |
| **Link back** | Each guide links to its parent hub |

Feature narratives belong in **`03-feature-guides/`**. Use **`02-recipes/`** for copy-paste examples and **`04-api-reference/`** only if you add terse API sheets — see [API reference hub](/docs#00-api-reference-overview).
