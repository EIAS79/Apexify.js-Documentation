**Advanced** here means **cross-cutting behaviour**: how pieces combine, performance intuition, and pointers ‚Äî **not** repeating full option matrices (those live in each linked **feature guide** from the **[Feature guides hub](/docs#feature-guides-hub)**).

For **what to ship** (social covers, Discord cards, chart PNGs, GIFs, ‚Ä¶) organized by feature, see **[What you can build](/docs#01-what-you-can-build)**.

---

## Composition model

Most flows are:

1. **`createCanvas`** ‚Üí base bitmap (+ backgrounds).  
2. **`createImage`** / **`createText`** / **`path2d.custom`** / **`path2d.draw`** ‚Üí layers on that buffer.  
3. **`save`** / **`outPut`** ‚Üí files or encoded strings ‚Äî see [Raster, batch & output](/docs#00-raster-batch-output-overview).

For **declarative** multi-layer stacks (charts, nested surfaces, one-shot PNG/GIF/video), use **[Scene overview](/docs#00-scene-composition)** (**`createScene`**, **`renderScene`**, **`renderSceneToGIF`**, **`renderSceneToVideoFrames`**).

**5.4+** adds reusable **templates** (**`{{placeholders}}`**, flex/grid **layout**), **`painter.assets`** for **`$name`** / **`$palette.key`**, **`painter.components`** (badge, progress bar, avatar, card, watermark ‚Üí **`SceneLayer[]`**), **`painter.plugins`** / **`painter.use`**, and **opt-in **`$`** resolution** on imperative paints and **batch / chain**. Those topics are written up in the **composition** chapter set (first file is the hub below).

Charts and GIF/video helpers either return buffers or orchestrate their own pipelines; each topic guide explains ordering.

## Performance intuition

- **`createImage`** filters often use **Sharp** (good for full-frame work).  
- **`pixels.getData`** / **`pixels.manipulate`** (or legacy names in older docs) are **CPU loops** ‚Äî best for small regions or custom logic ([Pixel data](/docs#08-pixel-data)).  
- **`batch`** runs independent jobs in **parallel** (`Promise.all`); **`chain`** runs **sequentially** and pipes buffers ‚Äî [Batch & chain](/docs#01-batch-and-chain).
