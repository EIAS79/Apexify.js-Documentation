# Recipes {#00-recipes-overview}

**Recipes** (`recipe-*.mdx` in this folder) are **goal-first walkthroughs**: what you want on disk **`/`** in an API response, prerequisites, steps, and a **full runnable snippet**. They sit **above** copy-paste noise and **below** option-by-option manuals.

**Feature guides** (`docs/03-feature-guides/`) explain **every flag**. **Recipes** explain **why this combination of APIs** solves one product-shaped problem.

## Folder conventions

| Rule | Why |
|------|-----|
| **One recipe per file** | Stable anchors; easier reviews |
| **Name** | **`recipe-<topic>.mdx`** (kebab-case) |
| **Structure** | Title → **Goal** → **Prerequisites** → **Steps** → **Full example** → **See also** |
| **Anchor** | **`{#recipe-<topic>}`** on the **`#`** title |
