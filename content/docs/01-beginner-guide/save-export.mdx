# Save & Export

## What this does

After painting buffers with **`createCanvas`**, **`createText`**, **`createImage`**, or **`createChart`**, you persist bytes to disk (or hand them to another service).

## When to use it

Final PNG/JPEG/WebP delivery, batch renders, CI-generated assets, or passing buffers into GIF/video pipelines.

## Quick example

<CodeSwitcher>
<ts>
```typescript
import { ApexPainter } from "apexify.js";
import fs from "fs";

const painter = new ApexPainter();

const { buffer } = await painter.createCanvas({
  width: 640,
  height: 360,
  colorBg: "#172554",
});

fs.writeFileSync("snapshot.png", buffer);
```
</ts>
<js>
```javascript
const { ApexPainter } = require("apexify.js");
const fs = require("fs");

(async () => {
  const painter = new ApexPainter();
  const { buffer } = await painter.createCanvas({
    width: 640,
    height: 360,
    colorBg: "#172554",
  });
  fs.writeFileSync("snapshot.png", buffer);
})();
```
</js>
</CodeSwitcher>

## How it works

1. **`fs.writeFileSync(path, buffer)`** is the simplest Node pattern — synchronous and predictable while learning.
2. Anything returned from Apexify render calls is a **`Buffer`** ready for **`writeFile`**.
3. For multiple outputs, reuse the painter and vary filenames (`frame-01.png`, …).

## Common changes

- Wrap writes in **`async`** functions when you switch to **`fs.promises.writeFile`** for parallelism.
- Prefix filenames with folders (`output/card.png`) — create folders first if needed.

## Common mistakes

- **Saving before awaiting** — finish **`await painter.create…`** before writing.
- **Overwriting blindly** — include timestamps or slugs when generating batches.
