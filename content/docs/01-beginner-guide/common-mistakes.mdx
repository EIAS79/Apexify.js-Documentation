## How it works

Each mistake pairs **symptom → fix** so you can correct course without undoing your whole project.

## Symptom → fix

### “It prints `[object Promise]` or TypeErrors inside Apex calls”

You forgot **`await`**. Every **`createCanvas`**, **`createImage`**, **`createText`**, **`createChart`** call returns a **Promise** — wrap usage in an **`async`** function.

### “My PNG is empty / unchanged after drawing”

You saved the **old buffer**. Always capture the **latest** return:

`buffer = await painter.createText(..., buffer)` — then save **`buffer`**.

### “Text is clipped or sits off-screen”

**`x` / `y`** plus **`textAlign` / `textBaseline`** describe **anchor points**, not automatic margins. Start centered using **`textAlign: "center"`**, **`textBaseline: "middle"`**, **`x = width / 2`**, **`y = height / 2`**.

### “Shapes overlap weirdly”

**Draw order = stacking order.** Earlier **`createImage`** calls sit underneath later ones.

### “Gradients look muddy”

Reduce **`noiseBg`** intensity or simplify **`gradientBg` stops** — contrast disappears when too many effects compete.

### “GIF or MP4 exports fail on my laptop”

Animated GIFs still need valid frames. **Video encoding** generally requires **FFmpeg installed** and available on your **`PATH`** — verify with your OS package manager before debugging code.

### “Batch jobs overwrite each other”

Generate **unique filenames** (slug + timestamp) or write into separate folders per job.

## Common changes

- Wrap experiments in **`try/catch`** so Apexify errors surface clearly.
- Log **`buffer.length`** after each draw to confirm growth.

## Common mistakes

Even this page has meta mistakes:

- **Skipping the beginner golden example** — confirm **[Your First Image](/docs#your-first-image)** works before layering complexity.
