## Imports

```typescript
import { ApexPainter, CanvasUtils, CanvasTypes } from 'apexify.js';
// Same helpers/types are also flat-exported from the barrel:
import type { CanvasConfig, ImageProperties } from 'apexify.js';
import { drawShape, url, base64 } from 'apexify.js';
```

| Export | Role | Source |
|--------|------|--------|
| **`ApexPainter`** | User façade — **`create*`** scene APIs, GIF, charts, video router (**`createVideo`**), batch/chain, standalone raster helpers, save/**`outPut`** | `lib/Canvas/ApexPainter.ts` |
| **`CanvasUtils`** | Namespace object re-exporting **`canvasUtils`** | `lib/Canvas/utils/canvasUtils.ts` |
| **`CanvasTypes`** | Types barrel (`CanvasConfig`, …) | `lib/Canvas/utils/types/index.ts` |
| **Flat `export *`** | Same symbols as **`CanvasUtils`** without nesting | `canvasUtils.ts` via `lib/index.ts` |

## Constructor & default serialization

```typescript
const painter = new ApexPainter({ type: 'buffer' }); // default
// type drives `outPut()` conversions: 'buffer' | 'base64' | 'dataURL' | 'blob' | 'url' | 'arraybuffer'
```

Most methods still return raw **`Buffer`** (PNG) regardless of **`type`**; **`outPut`** applies the constructor channel when you need a string/Blob/URL.

**Related:** [Save & output](/docs#save-and-output), [Batch & I/O hub](/docs#batch-chain-save-output).
