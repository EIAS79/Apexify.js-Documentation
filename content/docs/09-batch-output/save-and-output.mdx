# Save & output

Convert PNG **`Buffer`**s with **`outPut`** (base64, data URL, Blob, ArrayBuffer, **or hosted Imgur URL**) — or persist to disk with **`save`** / **`saveMultiple`** (Sharp-backed encoding).

Hub: [Batch, chain, save & output](/docs#batch-chain-save-output).

---

## How **`outPut`** works

1. **`createCanvas`**, **`createImage`**, **`createText`**, **`batch`**, etc. still give you a raw **PNG `Buffer`** (same regardless of ctor **`type`**).
2. Pass that **`Buffer`** to **`await painter.outPut(buffer)`**.
3. **`outPut`** reads **`new ApexPainter({ type: … })`** and returns the matching shape.

So **`type`** does **not** change what draw APIs return — it only configures **`outPut`**.

---

## Default output channel (`new ApexPainter`)

```typescript
const painter = new ApexPainter({ type: 'base64' });
```

| Constructor `type` | `await painter.outPut(buffer)` returns |
|--------------------|------------------------------------------|
| **`buffer`** (default) | Same **`Buffer`** (no-op pass-through) |
| **`base64`** | Base64 **`string`** |
| **`dataURL`** | **`data:image/png;base64,...`** **`string`** |
| **`blob`** | **`Blob`** |
| **`arraybuffer`** | **`ArrayBuffer`** view of the buffer |
| **`url`** | **HTTPS link **`string`** — uploads via **`url()`** in **`conversion.ts`** (Imgur API, bundled credentials) |

---

## `outPut(results: Buffer)`

**Requires** a **`Buffer`**. Uses **`this.format.type`** from the constructor.

**Throws:** wrong input → **`outPut: results must be a Buffer.`** · unknown **`type`** → lists supported kinds · wrapper → **`outPut failed: …`**.

### Example: base64

<CodeSwitcher>
<ts>
```typescript
import { ApexPainter } from 'apexify.js';

const painter = new ApexPainter({ type: 'base64' });

const { buffer } = await painter.createCanvas({
  width: 120,
  height: 120,
  colorBg: '#22c55e',
});

const payload = await painter.outPut(buffer);
console.log(typeof payload === 'string'); // true — base64 text
```
</ts>
<js>
```javascript
const { ApexPainter } = require('apexify.js');

(async () => {
  const painter = new ApexPainter({ type: 'base64' });
  const { buffer } = await painter.createCanvas({
    width: 120,
    height: 120,
    colorBg: '#22c55e',
  });
  const payload = await painter.outPut(buffer);
  console.log(typeof payload === 'string');
})();
```
</js>
</CodeSwitcher>

## Example: **`outPut` → Imgur URL {#save-and-output-imgur-url}

Use this when you want a **public URL** for a generated image (Discord embeds, quick shares, prototypes) **without** implementing upload yourself. The library calls **`url(buffer)`** (**`lib/Canvas/utils/general/conversion.ts`**) which uses the **Imgur API** with credentials shipped in that module.

**Requirements:** outbound **HTTPS** at runtime; Imgur must accept the upload (limits / account status per Imgur).

<CodeSwitcher>
<ts>
```typescript
import { ApexPainter } from 'apexify.js';

const painter = new ApexPainter({ type: 'url' });

const { buffer } = await painter.createCanvas({
  width: 640,
  height: 360,
  colorBg: '#0f172a',
});

const imageUrl = await painter.outPut(buffer);
console.log(imageUrl);
```
</ts>
<js>
```javascript
const { ApexPainter } = require('apexify.js');

(async () => {
  const painter = new ApexPainter({ type: 'url' });
  const { buffer } = await painter.createCanvas({
    width: 640,
    height: 360,
    colorBg: '#0f172a',
  });
  const imageUrl = await painter.outPut(buffer);
  console.log(imageUrl);
})();
```
</js>
</CodeSwitcher>

**Same pattern after any pipeline:** `createImage`, `createText`, **`batch`**/**`chain`** results, **`GIF`** buffers — anything that ends as **`Buffer`**, pass into **`outPut`**.

```typescript
const painter = new ApexPainter({ type: 'url' });
const { buffer: scene } = await painter.createCanvas({ width: 400, height: 200, colorBg: '#ffffff' });
const png = await painter.createText(
  { text: 'Hello', x: 80, y: 120, font: { size: 48, family: 'Arial' } },
  scene,
);
const link = await painter.outPut(png);
```

If you need **your own** Imgur app or a different host, use **`type: 'base64'`** / **`dataURL`** and upload with your SDK, or maintain a fork that swaps **`url()`** implementation — the **`outPut`** switch stays the same (`case 'url'`).

## `save(buffer, options?)`

Writes **`./ApexPainter_output`** by default (creates directories when **`createDirectory`** is **`true`**).

### `SaveOptions`

| Field | Default | Notes |
|-------|---------|--------|
| **`directory`** | **`./ApexPainter_output`** | Target folder |
| **`filename`** | auto | If set without extension, **`.${format}`** appended |
| **`format`** | **`png`** | **`jpg`/`jpeg`**, **`webp`**, **`avif`**, **`gif`** |
| **`quality`** | **90** | JPEG/WebP/AVIF |
| **`createDirectory`** | **`true`** | **`fs.mkdirSync`** recursive |
| **`naming`** | **`timestamp`** | **`counter`**, **`custom`** |
| **`counterStart`** | **1** | Stored on options (counter filenames still advance **`painter`**’s internal counter) |
| **`prefix`**, **`suffix`** | **`''`** | Applied to generated names |
| **`overwrite`** | **`false`** | When **`false`**, appends **`_1`**, **`_2`**, … if file exists |

Non-PNG formats run through **Sharp** before **`writeFileSync`**.

<CodeSwitcher>
<ts>
```typescript
import { ApexPainter } from 'apexify.js';

const painter = new ApexPainter();
const { buffer } = await painter.createCanvas({ width: 640, height: 360, colorBg: '#111' });

const saved = await painter.save(buffer, {
  directory: './exports',
  filename: 'hero.webp',
  format: 'webp',
  quality: 88,
});

console.log(saved.path, saved.size);
```
</ts>
<js>
```javascript
const { ApexPainter } = require('apexify.js');

(async () => {
  const painter = new ApexPainter();
  const { buffer } = await painter.createCanvas({ width: 640, height: 360, colorBg: '#111' });
  const saved = await painter.save(buffer, {
    directory: './exports',
    filename: 'hero.webp',
    format: 'webp',
    quality: 88,
  });
  console.log(saved.path, saved.size);
})();
```
</js>
</CodeSwitcher>

### `SaveResult`

**`path`**, **`filename`**, **`size`**, **`format`**.

## `saveMultiple(buffers, options?)`

Saves **each** buffer sequentially with shared **`SaveOptions`**. Requires a **non-empty** array.

<CodeSwitcher>
<ts>
```typescript
const painter = new ApexPainter();

const shots = await painter.batch([
  { type: 'canvas', config: { width: 200, height: 200, colorBg: '#ef4444' } },
  { type: 'canvas', config: { width: 200, height: 200, colorBg: '#3b82f6' } },
]);

const files = await painter.saveMultiple(shots, {
  directory: './shots',
  naming: 'counter',
  prefix: 'frame-',
  format: 'png',
});

files.forEach((f) => console.log(f.filename));
```
</ts>
<js>
```javascript
const painter = new ApexPainter();

(async () => {
  const shots = await painter.batch([
    { type: 'canvas', config: { width: 200, height: 200, colorBg: '#ef4444' } },
    { type: 'canvas', config: { width: 200, height: 200, colorBg: '#3b82f6' } },
  ]);
  const files = await painter.saveMultiple(shots, {
    directory: './shots',
    naming: 'counter',
    prefix: 'frame-',
    format: 'png',
  });
  files.forEach((f) => console.log(f.filename));
})();
```
</js>
</CodeSwitcher>

**Throws:** **`saveMultiple: buffers must be a non-empty array.`**

---

## `resetSaveCounter()`

Sets the internal **`saveCounter`** back to **1** — useful before another **`save`** / **`saveMultiple`** run with **`naming: 'counter'`** so numbering restarts cleanly.
