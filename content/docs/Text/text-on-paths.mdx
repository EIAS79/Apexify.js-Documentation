# Curved text, arcs, and 2D path layouts

Apexify.js **v5.3.x** (current npm `dist`) renders **bent / circular-arc text** through the **`textOnCurve`** option on `createText`. Each glyph is placed on a circle with correct tangential rotation so lettering stays readable along the arc.

<Alert type="warning" title="Docs vs published package">
Older versions of this page described **`textOnPath: true`** with a **`path: { type: 'line' | 'bezier' | … }`** API. That surface **does not appear** in the published **`apexify.js` 5.3.12** bundle (`EnhancedTextRenderer` only branches on **`textOnCurve`**). The examples below match **`dist/cjs/Canvas/utils/Texts/enhancedTextRenderer.js`**. For arbitrary polylines or Bézier guides, use **[Path2D](/docs#path2d-api)** to draw the guide and place straight text, or post-process glyph positions yourself until a path-follow API ships.
</Alert>

## Overview

| Capability | In npm 5.3.x? | How |
|------------|---------------|-----|
| **Circular arc (“smile” / badge text)** | Yes | `textOnCurve: { sweepAngle, radius?, up? }` |
| **Multi-line arcs** | Yes | Use `\n` in `text`; each line gets its own arc, stacked by `lineHeight` |
| **Per-glyph effects on curves** | Yes | `stroke`, `shadow`, `glow`, `highlight`, `underline`, … run in local tangent space |
| **Straight line / Bézier / quadratic “text on path”** | Not in current `dist` | Draw with **`drawPath`** / geometry docs; no built-in glyph sampler for those curves yet |

---

## `textOnCurve` reference

When **`textOnCurve`** is set, the renderer **ignores the normal straight-line layout** for that object (it does **not** combine arc mode with `maxWidth` word-wrap—the curved branch runs first).

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| **`sweepAngle`** | `number` (degrees) | _required_ | Total angle subtended by the string along the arc. **`≤ 0`** or **`≥ 360`** falls back to **straight** text at `(x, y)`. |
| **`radius`** | `number` | `totalWidth / θ` | Circle radius in px, where `θ` is `sweepAngle` in radians. Smaller `sweepAngle` → tighter curve if radius is auto. |
| **`up`** | `boolean` | `true` | **`true`**: arc center is **below** the anchor (common “smile” / top-of-badge). **`false`**: arc opens the other way (frown / bottom band). |

### Anchor point `(x, y)`

`(x, y)` is the **baseline anchor** for the line: the implementation treats it as the **horizontal center** of the string on the arc (middle glyph). The circle center is derived from `y`, `radius`, and `up` (see renderer: `cy = up ? y + R : y - R`).

### Letter-spacing and kerning

`letterSpacing` / `wordSpacing` are applied via the canvas font; arc positions use **measured cumulative widths** along the string so extra space is distributed along the arc with the glyphs.

## Examples

### Smile arc (default `up: true`)

<CodeSwitcher>
<ts>
```typescript
import { ApexPainter } from 'apexify.js';
import fs from 'fs';

const painter = new ApexPainter();

const canvas = await painter.createCanvas({
  width: 900,
  height: 400,
  colorBg: '#0f172a',
});

const buf = await painter.createText(
  {
    text: 'APEXIFY · CURVED TYPE',
    x: 450,
    y: 260,
    fontSize: 36,
    fontFamily: 'Arial',
    color: '#e2e8f0',
    textAlign: 'center',
    textBaseline: 'middle',
    textOnCurve: {
      sweepAngle: 140,
      up: true,
    },
    stroke: { color: '#38bdf8', width: 1 },
  },
  canvas.buffer
);

fs.writeFileSync('arc-smile.png', buf);
```
</ts>
<js>
```javascript
const { ApexPainter } = require('apexify.js');
const fs = require('fs');

const painter = new ApexPainter();

async function main() {
  const canvas = await painter.createCanvas({
    width: 900,
    height: 400,
    colorBg: '#0f172a',
  });

  const buf = await painter.createText(
    {
      text: 'APEXIFY · CURVED TYPE',
      x: 450,
      y: 260,
      fontSize: 36,
      fontFamily: 'Arial',
      color: '#e2e8f0',
      textAlign: 'center',
      textBaseline: 'middle',
      textOnCurve: { sweepAngle: 140, up: true },

}

main();
```

### Straight fallback (`sweepAngle` out of range)

If `sweepAngle` is **`≤ 0`** or **`≥ 360`**, the renderer draws a **normal straight** line at `(x, y)` (same as omitting `textOnCurve`).

---

## Combining with global `rotation`

`rotation` on the text object is applied **before** arc layout (canvas translate/rotate around `(x, y)`). Use **either** a strong arc **or** a large global rotation—not both—until you preview the interaction, since both affect the final basis.

---

## “True” 2D path text (lines, Béziers, loops)

1. **Draw the path** with **`createPath2D`** + **`drawPath`** (stroke/fill) so users see the guide. See **[Path2D API](/docs#path2d-api)**.
2. **Sample the path** in your own code (small steps along length), and for each sample call **`createText`** with a **single character** (or short substring) and **`rotation`** set from the tangent angle. That reproduces classic “text on path” without engine support.
3. **Hit-testing** along custom shapes uses **`isPointInPath`** / **`isPointInRegion`** — see **[Hit detection](/docs#hit-detection-api)**.
